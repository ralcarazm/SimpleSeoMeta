# Simple SEO Meta

**Simple SEO Meta** is an Omeka Classic plugin that adds configurable SEO metadata to public pages. It provides global SEO defaults for the site and individual metadata overrides for **Simple Pages**.

The plugin can output and manage:

- the HTML `<title>` element;
- `<meta name="description">`;
- `<meta name="robots">`;
- canonical URLs;
- Open Graph metadata;
- Twitter/X Card metadata.

It also includes localization support and a Catalan translation for `ca_ES`.

## Description

Simple SEO Meta allows administrators to define default SEO metadata for an Omeka Classic site and override those values on individual Simple Pages.

The plugin is designed for small and medium-sized Omeka Classic projects that need basic but explicit control over the metadata that appears in the public `<head>` of the site.

It is especially useful for:

- improving control over page titles and search snippets;
- defining canonical URLs;
- controlling crawler behaviour with robots directives;
- improving link previews in social platforms and messaging applications;
- adding per-page SEO metadata to Simple Pages without editing the theme.

## Features

### Global SEO defaults

From the plugin configuration screen, administrators can define global fallback values for the whole site.

Global settings include:

- title rewriting;
- title template;
- default page title;
- default meta description;
- canonical URL output;
- query string removal from canonical URLs;
- global meta robots directives;
- additional robots directives;
- Open Graph defaults;
- Twitter/X Card defaults.

### Per-Simple Page overrides

When the **SimplePages** plugin is installed and enabled, Simple SEO Meta adds a **Simple SEO Meta** accordion to the Simple Pages add/edit form.

Each Simple Page can override the global defaults for:

- SEO title;
- meta description;
- canonical URL;
- meta robots;
- additional robots directives;
- Open Graph metadata;
- Twitter/X Card metadata.

If a field is left empty, the plugin uses the global setting or an automatic fallback.

### Automatic fallbacks

When no custom metadata is provided, the plugin attempts to infer useful values from the current page.

For Simple Pages:

- page title is used as the fallback title;
- page text is used to generate a fallback description.

For item pages:

- Dublin Core Title is used as the fallback title;
- Dublin Core Description is used as the fallback description;

For collection pages:

- Dublin Core Title is used as the fallback title;
- Dublin Core Description is used as the fallback description.

For other pages:

- the site title and the configured global defaults are used where possible.

## Editable SEO fields

### Basic SEO fields

| Field | Global default | Per-Simple Page override | Output |
|---|---:|---:|---|
| Title rewriting | Yes | No | Replaces the first public `<title>` tag generated by the theme |
| Title template | Yes | No | Controls how inferred titles are formatted |
| Default page title | Yes | No | Used when no page title can be inferred |
| SEO title | No | Yes | Used for the public `<title>` |
| Meta description | Yes | Yes | `<meta name="description">` |
| Canonical URL | Yes | Yes | `<link rel="canonical">` |

### Meta robots fields

| Field | Global default | Per-Simple Page override | Output |
|---|---:|---:|---|
| Base directive | Yes | Yes | `<meta name="robots">` |
| Custom robots value | Yes | Yes | Custom value for `<meta name="robots">` |
| `nosnippet` | Yes | Yes | Additional robots directive |
| `indexifembedded` | Yes | Yes | Additional robots directive |
| `notranslate` | Yes | Yes | Additional robots directive |
| `noimageindex` | Yes | Yes | Additional robots directive |
| `noarchive` | Yes | Yes | Additional robots directive |
| `nocache` | Yes | Yes | Additional robots directive |
| `nositelinkssearchbox` | Yes | Yes | Additional robots directive |
| `max-snippet` | Yes | Yes | Additional robots directive |
| `max-image-preview` | Yes | Yes | Additional robots directive |
| `max-video-preview` | Yes | Yes | Additional robots directive |
| `unavailable_after` | Yes | Yes | Additional robots directive |

Available base directives:

- no robots meta tag;
- `all`;
- `index,follow`;
- `index,nofollow`;
- `noindex,follow`;
- `noindex,nofollow`;
- `noindex`;
- `nofollow`;
- `none`;
- custom value.

Important distinction:

- leaving the global robots directive empty means that no robots meta tag is output; in normal crawler behaviour this is usually equivalent to `index,follow`;
- `all` means no restrictions and is also equivalent in practice to `index,follow`;
- `none` is a shortcut for `noindex,nofollow`.

Available `max-image-preview` values:

- no output;
- `none`;
- `standard`;
- `large`.

### Open Graph fields

| Field | Global default | Per-Simple Page override | Output |
|---|---:|---:|---|
| Enable Open Graph | Yes | No | Enables/disables Open Graph output |
| Default `og:type` | Yes | Yes | `<meta property="og:type">` |
| `og:locale` | Yes | No | `<meta property="og:locale">` |
| Default image URL | Yes | Yes | `<meta property="og:image">` |
| Default image alt text | Yes | Yes | `<meta property="og:image:alt">` |
| Open Graph title | No | Yes | `<meta property="og:title">` |
| Open Graph description | No | Yes | `<meta property="og:description">` |

Generated Open Graph tags may include:

```html
<meta property="og:title" content="...">
<meta property="og:description" content="...">
<meta property="og:type" content="website">
<meta property="og:url" content="...">
<meta property="og:site_name" content="...">
<meta property="og:locale" content="...">
<meta property="og:image" content="...">
<meta property="og:image:alt" content="...">
```

### Twitter/X Card fields

| Field | Global default | Per-Simple Page override | Output |
|---|---:|---:|---|
| Enable Twitter Cards | Yes | No | Enables/disables Twitter/X Card output |
| Card type | Yes | Yes | `<meta name="twitter:card">` |
| `twitter:site` | Yes | No | `<meta name="twitter:site">` |
| `twitter:creator` | Yes | No | `<meta name="twitter:creator">` |
| Twitter/X title | No | Yes | `<meta name="twitter:title">` |
| Twitter/X description | No | Yes | `<meta name="twitter:description">` |
| Twitter/X image URL | No | Yes | `<meta name="twitter:image">` |
| Twitter/X image alt text | No | Yes | `<meta name="twitter:image:alt">` |

Available card types:

- `summary`;
- `summary_large_image`.

Generated Twitter/X Card tags may include:

```html
<meta name="twitter:card" content="summary_large_image">
<meta name="twitter:site" content="@your_site">
<meta name="twitter:creator" content="@creator">
<meta name="twitter:title" content="...">
<meta name="twitter:description" content="...">
<meta name="twitter:image" content="...">
<meta name="twitter:image:alt" content="...">
```

## Installation

1. Download the plugin.
2. Make sure the folder is named exactly:

```text
SimpleSeoMeta
```

3. Copy the folder into the Omeka Classic plugins directory:

```text
/plugins/SimpleSeoMeta
```

4. Log in to the Omeka Classic admin area.
5. Go to:

```text
Plugins
```

6. Find **Simple SEO Meta** and click **Install**.
7. Click **Configure** to define the global SEO defaults.

The final folder structure should look like this:

```text
plugins/
└── SimpleSeoMeta/
    ├── plugin.ini
    ├── plugin.php
    ├── SimpleSeoMetaPlugin.php
    ├── css/
    │   └── simple-seo-meta-admin.css
    ├── javascripts/
    │   └── simple-seo-meta-admin.js
    └── languages/
        ├── ca_ES.po
        ├── ca_ES.mo
        └── template.pot
```

## Upgrading

1. Back up your Omeka Classic installation and database.
2. Replace the old `SimpleSeoMeta` folder with the new one.
3. Go to the Omeka Classic admin area.
4. Open:

```text
Plugins
```

5. Click **Upgrade** next to **Simple SEO Meta**, if Omeka shows the upgrade option.

The plugin stores per-page metadata in its own database table. The upgrade process ensures that this table exists.

## Configuration

Go to:

```text
Admin > Plugins > Simple SEO Meta > Configure
```

The configuration screen contains four main areas.

### Global SEO defaults

Use this section to configure:

- whether the plugin should rewrite the public `<title>` tag;
- the default title template;
- the default page title;
- the default meta description;
- canonical URL output;
- whether query strings should be removed from canonical URLs.

The title template supports these variables:

```text
{page_title}
{site_title}
```

Example:

```text
{page_title} | {site_title}
```

### Global meta robots defaults

Use this section to configure the default robots behaviour for the site.

You can choose a base directive, define a custom robots value and add advanced directives such as:

- `nosnippet`;
- `indexifembedded`;
- `notranslate`;
- `noimageindex`;
- `noarchive`;
- `nocache`;
- `nositelinkssearchbox`;
- `max-snippet`;
- `max-image-preview`;
- `max-video-preview`;
- `unavailable_after`.

### Open Graph

Use this section to enable Open Graph metadata and define default values such as:

- `og:type`;
- `og:locale`;
- default image URL;
- default image alt text.

### Twitter Cards

Use this section to enable Twitter/X Card metadata and define:

- default card type;
- `twitter:site`;
- `twitter:creator`.

## Editing metadata for individual Simple Pages

When the SimplePages plugin is installed, a **Simple SEO Meta** accordion appears in the Simple Pages add/edit form.

To edit metadata for a Simple Page:

1. Go to:

```text
Admin > Simple Pages
```

2. Add or edit a page.
3. Open the **Simple SEO Meta** accordion.
4. Fill in the fields you want to override.
5. Leave fields empty to use global defaults and automatic fallbacks.
6. Save the page.

The per-page form is intentionally collapsed by default so that the standard Simple Pages form remains uncluttered.

## Output

Simple SEO Meta writes metadata into the public `<head>` of Omeka Classic pages.

A typical output may look like this:

```html
<!-- Simple SEO Meta -->
<meta name="description" content="...">
<meta name="robots" content="index, follow">
<link rel="canonical" href="https://example.org/page/example">
<meta property="og:title" content="...">
<meta property="og:description" content="...">
<meta property="og:type" content="website">
<meta property="og:url" content="https://example.org/page/example">
<meta property="og:site_name" content="...">
<meta property="og:image" content="https://example.org/files/example.jpg">
<meta property="og:image:alt" content="...">
<meta name="twitter:card" content="summary_large_image">
<meta name="twitter:title" content="...">
<meta name="twitter:description" content="...">
<meta name="twitter:image" content="https://example.org/files/example.jpg">
<meta name="twitter:image:alt" content="...">
<!-- /Simple SEO Meta -->
```

If title rewriting is enabled, the plugin also replaces the first public `<title>` tag generated by the theme.

## Localization

Simple SEO Meta supports localization through Omeka Classic translation files.

Included languages:

- `ca_ES` — Catalan.

The translation files are stored in:

```text
languages/
├── ca_ES.po
├── ca_ES.mo
└── template.pot
```

To use the Catalan translation, configure Omeka Classic with:

```ini
locale.name = "ca_ES"
```

in:

```text
application/config/config.ini
```

Additional translations can be created from:

```text
languages/template.pot
```

## Database

The plugin creates a dedicated database table for per-record metadata.

The table stores:

- record type;
- record ID;
- metadata values as JSON;
- insertion timestamp;
- update timestamp.

The table is created on install or upgrade.

On uninstall, the plugin removes its options and drops its metadata table.

## Notes and limitations

- Per-page manual editing is currently implemented for **Simple Pages**.
- Item and collection pages can receive automatic metadata based on existing Dublin Core metadata, but this version does not add manual SEO fields to the item or collection edit forms.
- The plugin rewrites the first `<title>` tag in the public HTML output when title rewriting is enabled.
- The plugin does not guarantee how search engines or social platforms will display the metadata.

## Uninstallation

Uninstalling the plugin removes:

- plugin configuration options;
- stored per-page SEO metadata;
- the plugin metadata table.

Back up the database before uninstalling if you need to preserve existing SEO metadata.
